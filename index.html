Why Atez exists

Security skills only mean something if they're earned honestly.

We run labs, CTFs, and study groups so members can learn how systems break — legally, safely, and with people who'll explain the "why," not just the "how."

SCOPE

Consent, always

Every exercise runs on infrastructure we own or that's explicitly built for training. No live targets, no exceptions.

METHOD

Understand before you exploit

We teach the underlying mechanics — protocols, memory, logic flaws — so members can reason about new problems, not just replay steps.

PURPOSE

Defense is the point

Every offensive technique we cover pairs with the detection and hardening side. Breaking things responsibly is how you learn to protect them.

The Range

Six lanes. All sandboxed.

Each lane is a self-contained environment members spin up on demand — vulnerable-by-design boxes, isolated networks, and snapshots that reset after every session. Below are the categories members work in and the established, industry-standard tools each lane is built around.

LANE 01 — RECON Sandboxed

Reconnaissance & Mapping

Enumerate hosts, services, and attack surface inside a closed lab network before touching anything else.

Nmap Wireshark Amass
LANE 02 — WEB Sandboxed

Web Application Security

Work through injection, auth, and logic flaws on deliberately vulnerable apps built for practice.

Burp Suite CE OWASP ZAP DVWA
LANE 03 — CRYPTO Sandboxed

Cryptography & Encoding

Break intentionally weak ciphers and encodings to understand how and why real implementations fail.

CyberChef OpenSSL dCode
LANE 04 — FORENSICS Sandboxed

Digital Forensics

Trace incidents through disk images, memory captures, and packet dumps from staged scenarios.

SIFT Workstation Volatility Autopsy
LANE 05 — REVERSING Sandboxed

Reverse Engineering

Pull apart binaries built for practice to understand compiled logic, obfuscation, and protection schemes.

Ghidra radare2 x64dbg
LANE 06 — DEFENSE Sandboxed

Blue Team & Detection

Build detections for the attacks run in the other lanes — logging, alerting, and incident response drills.

Elastic Security Zeek Security Onion
How it works

From "curious" to "capable," on a real path.

No prior experience required — every lane has an entry point for beginners and a harder track once you're ready.

STEP 01

Join the Discord

That's the front door. Introduce yourself, pick a lane, and get access to our lab guides and scheduled sessions.

STEP 02

Spin up a sandbox

Each exercise runs in an isolated VM or container built specifically for it. Nothing you touch is a production system, ever.

STEP 03

Work the lane with others

Pair up in voice, compare approaches, and get unstuck faster than you would solo. Mentors sit in on most sessions.

STEP 04

Take it to a CTF

Once you're comfortable, we run and compete in capture-the-flag events as a team — the legal, structured version of the real thing.
